# Hermes Agent Capability Audit

Scope: compare `D:\pro_sunner\demo_vscode\hermes-agent` with SynthChat's current Rust/Tauri agent implementation. This file is a living completion audit, not a feature proposal.

## MVP Closure Scope

As of 2026-06-13, the short-term closure target is a stable SynthChat desktop agent MVP rather than full Hermes 1:1 parity. The MVP includes the native desktop agent loop, queued runs, approval/abort controls, file/terminal/browser/media/memory tools, core dashboard API adapters, Kanban/task visibility, provider/auth/status diagnostics, and the bounded dashboard plugin runner with common FastAPI/Starlette/Pydantic compatibility.

Deferred non-core Hermes parity: embedding the full long-lived Hermes FastAPI dashboard host, arbitrary Python `plugin_api.py` ASGI route ownership, arbitrary plugin WebSocket handler execution, full Python `PluginManager` / live `PluginContext` object identity, complete external Hermes daemon lifecycle, deep platform runtimes such as Matrix Olm/Megolm, Teams SDK object lifecycle, Google Meet realtime audio/browser orchestration, SimpleX lifecycle, and live third-party E2E proof for every optional provider. These remain tracked as future parity work and are not required for the current MVP validation gate.

Minimal validation gate for this closure: formatting must pass, touched-file whitespace checks must pass, and focused tests covering the recently changed compatibility surface must pass. Full `cargo test` is intentionally deferred until final release hardening.
